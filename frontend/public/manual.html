01시스템 개요

Dental Wave Viz는 치은 부위에 30 kHz 탄성파를 가했을 때의 파동 전파 결과를 분석하여 치은 병변 후보 영역을 추정하는 비침습 스크리닝 워크플로우를 의사가 시각적으로 검토할 수 있도록 만든 데모 콘솔입니다.

본 시스템이 보여주는 것

  • 3D 구강 구조 모델 위에서의 파동장 전파 양상
  • 수신기 100개에서 기록된 시계열(시지모그램)
  • 시간 누적 에너지 분포에 기반한 후보 영역 스크리닝
  • Bayesian 역산을 통한 병변 위치 사후분포
  • 참값 대비 추정값의 정량 오차 (시뮬레이션 데이터에 한함)

본 시스템이 보여주지 않는 것

  • 실제 환자의 임상 영상 (CBCT/STL 합성 시뮬레이션 데이터 사용)
  • 치주낭 깊이(PPD), 부착 소실(CAL), 출혈(BoP) 등 기존 임상 지표
  • 병리학적 확진 결과
중요 · 면책 본 시스템은 시뮬레이션 데이터만을 이용한 PoC(Proof-of-Concept) 단계의 데모입니다. 모든 환자 / 스캔 / 병변 정보는 합성된 가상의 케이스이며, 표시되는 결과를 실제 임상 의사결정의 단일 근거로 사용해서는 안 됩니다.

구간최근 심각도의미

04스캔 분석 화면 구성

환자 상세에서 스캔 row를 클릭하면 분석 화면이 열립니다. 화면은 3열 구조입니다.

왼쪽 사이드바 — 스캔 메타정보

  • 환자 이름 / 스캔 ID / 스캔 일자
  • 시나리오 배지 (정상 / 염증 70% / 염증 80%)
  • 음향 파라미터 — 주파수, 신호 형태, 수신기 수, 해상도
  • 시나리오 — 현재 케이스 라벨
  • 주요 액션 버튼 3개:
    • 파이프라인 스토리 — 6단계 워크스루 화면
    • 케이스 비교 — 다른 시나리오와 나란히 비교
    • 보고서 PDF — 1페이지 영어 임상 양식 다운로드
  • 분석 파이프라인 6단계 요약 + 자세히 보기 링크

중앙 — 분석 워크스페이스

위에서 아래로 4개 영역으로 나뉩니다.

  • 케이스 토글 — baseline / position / size 시나리오 선택 버튼
  • KPI 스트립 — 5개 정량 지표 카드 (5장 참조)
  • 시간축 스크러버 — wavefield + seismogram 동기 제어
  • 6개 분석 패널 — 3행 그리드 (7장 참조)

오른쪽 사이드바 — 탐지 결과

  • 심각도 바 — 0–100% 시각화
  • 후보 수신기 번호, RMS 잔차, 추정 좌표(mm), 모델 빌드 버전
  • 역산 결과 요약 — true / mode / best, 거리 |Δ|
  • 권고 액션 — 심각도에 따른 자동 분류된 다음 단계
  • 의사 코멘트 — 진단 소견 입력 (doctor 계정만)
  • 재계산 버튼 (우상단)

05KPI 지표 읽기

중앙 워크스페이스 상단에 표시되는 5개 카드입니다. 각 지표는 임계치를 넘으면 색이 바뀝니다.

지표의미좋음 ✓주의 ⚠나쁨 ✗
Localization err 역산 추정 위치 ↔ 참값 거리 (mode − GT, voxel) ≤ 3 vox 3–6 vox > 6 vox
Lesion severity 병변 가능성 점수 (0% 정상 → 100% 확실) < 50% 50–80% ≥ 80%
MCMC R̂ 체인 수렴 진단 (split-half 분산 비) ≤ 1.05 1.05–1.10 > 1.10
ESS 유효 표본 크기 (고유 posterior state 수 / 1000) ≥ 200 100–200 < 100
Best misfit 관측 데이터와 최적 모델의 잔차 (작을수록 정합 우수) 절대값보다 케이스 간 상대 비교가 의미 있음
읽는 법 요약 Lesion severity가 높으면서 동시에 R̂이 낮고(≤1.05) ESS가 크면(≥200), 알고리즘이 높은 신뢰도로 병변을 잡았다고 해석할 수 있습니다. 반대로 severity가 높아도 R̂이 1.1 이상이면 MCMC가 충분히 수렴하지 않았으므로 결과를 보수적으로 해석해야 합니다.

06시간축 스크러버 사용법

KPI 스트립 바로 아래의 시간 슬라이더는 wavefield snapshot과 seismogram gather의 노란 가로선을 동시에 제어합니다.

구성

  • ▶ play / ❚❚ pause — 자동 진행 토글
  • 슬라이더 — 9개 시간 단계 (0–67500 step)
  • step / μs 표시 — 현재 단계와 마이크로초 환산값
  • 0.5× / 1× / 2× / 4× — 자동 진행 속도

이렇게 쓰면 좋습니다

1
play 클릭 → 파동 전파 양상을 동영상처럼 관찰
2
특정 시점이 흥미로우면 슬라이더를 직접 드래그하여 고정
3
seismogram gather의 가로 노란선이 wavefield와 동기 이동 — 어떤 시각의 신호가 어떤 수신기에 도달하는지 직관적으로 매핑

076개 분석 패널 사용법

7.1 Wavefield snapshot

3D 표면 그래프로 표시된 파동장 한 컷. 진폭이 곧 z축 고도이며, 색상은 SEISMIC 컬러맵 (음=파랑, 양=빨강).

  • 마우스 드래그: 회전 · 스크롤: 줌 · Shift+드래그: 팬
  • tissue 토글: 치아(불투명) · 골(반투명 적색) · 치은(연분홍) mesh 오버레이 ON/OFF
  • 카메라 프리셋: 8장 참조
  • 시간 스크러버로 다음 step으로 이동

7.2 Seismogram gather

가로축이 수신기 번호 (0–99), 세로축이 시간 (μs), 색이 정규화된 진폭. 노란 가로선이 현재 시간을 표시.

  • 대각선 패턴은 파면이 수신기 배열을 따라 전파되는 모습
  • 특정 수신기 컬럼에 비대칭적인 강한 신호가 나타나면 그 부근에 산란체가 존재할 가능성

7.3 GT vs Estimate · 3D overlay

참값(GT)과 posterior 추정의 시각적 비교. 3D 씬에는:

  • 녹색 + — Ground truth (참 병변 중심)
  • 자홍 ◆ — Posterior mean (사후 평균 추정)
  • 시안 mesh — 95% credible region (mean ± 1.96σ ellipsoid)
  • 노란 선 — error vector (GT → 추정)

아래 strip에 GT / 추정 / Δx · Δy · Δz / σ / |err| 정량값이 표시됩니다.

7.4 Energy profile

수신기별 누적 에너지를 1D 곡선으로 표시. 노란 점이 peak 수신기. 가로 드래그로 영역을 선택하면 해당 수신기들이 다른 패널에서 highlight (예: velocity slice).

7.5 A/B wipe · GT ↔ posterior

병변 삽입 전/후 조직 분류 맵을 좌우 와이프로 비교.

  • 슬라이더 드래그 — wipe 위치를 0–100% 사이로 조절
  • GT only 버튼 — 좌측 (염증 비포함) 풀스크린
  • Posterior 버튼 — 우측 (염증 자홍색 노출) 풀스크린
  • 차이 영역이 자홍색으로 나타나는 곳이 모델이 가정한 병변 위치

7.6 Screening surface

모든 시간 step의 에너지를 누적한 평면 heatmap. 97.5 percentile 이상 voxel을 hotspot로 분류, 헤더에 voxel 수 표시.

7.7 MCMC inversion (큰 패널)

아래쪽 풀폭 패널. 좌측 3D scatter + 우측 misfit 곡선.

좌측 3D scatter — 표시되는 것

  • 치아·골·치은 mesh (배경, 매우 옅음)
  • 분포 클라우드 — 방문 횟수가 많을수록 크고 진한 색
  • mode (시안 ⭕) — 최빈값
  • MAP (녹색 ◇) — 최소 misfit 모델
  • GT (빨간 ✚) — 참값
  • walk (노란 line) — iteration 순서대로 연결한 체인 경로

컨트롤

  • Play / Pause + 1×–10× 배속 + scrub 슬라이더
  • 각 마커는 개별 토글 가능 (mode / best / walk)
  • 우측 misfit 곡선: log scale, 최소값에 녹색 점
  • 완료 시 정량 지표 표 (각 파라미터의 true / mode / best와 거리)
  • accept 비율 표시 (20–60%가 건강한 Metropolis 체인)

08카메라 프리셋

3D 패널 헤더의 작은 토글로 표준 치과 시야로 즉시 전환할 수 있습니다.

버튼각도임상적 의미
3⁄4비스듬한 회전 뷰기본 — 전체 구조 조망
Occlusal위에서 아래 내려다보기교합면 (bite view) — 치아 배열을 위에서
Buccal볼 쪽에서 측면외측 시야 — 협면 노출
Lingual혀 쪽에서 측면내측 시야 — 설면 노출

적용 패널: Wavefield snapshot, GT vs Estimate, MCMC inversion (3D 씬 모두).

09케이스 비교 모드

좌측 사이드바의 케이스 비교 또는 좌측 네비게이션의 Compare를 통해 진입합니다.

구성

  • 상단: 좌(A) 와 우(B) 케이스 선택 드롭다운
  • 상단 우측: 비교할 패널 종류 선택 (Seismogram / Screening / Energy / Velocity / Snapshot)
  • 본문: 선택된 케이스 2개의 같은 패널을 나란히 표시

활용 예

  • 같은 환자의 시간 경과 비교: 같은 환자 스캔이 여러 건이면 변화 추적
  • 다른 시나리오 비교: baseline ↔ position shift ↔ size change 의 wave signature 차이 확인

10파이프라인 스토리 모드

분석 알고리즘의 6단계를 단계별 카드로 보여주는 walkthrough 화면입니다.

  1. CBCT + STL 정합 — 그레이스케일 단면 + 정합 crosshair
  2. 치아·조직 분할 (4-class) — bg → 치은 → 골 → 치아 순차 노출
  3. 병변 삽입 — 풀 라벨 + 펄싱 lesion 마커
  4. 탄성파 전파 시뮬레이션 — wavefield + seismogram 동시
  5. 저속 성분 + 후보 영역 — screening + energy
  6. 국소 역산 (MCMC) — 풀 inversion pane

사용법

  • 상단 step 레일에서 단계를 직접 클릭하거나 ← → 버튼으로 이동
  • 자동재생 버튼 → 6초마다 다음 단계로
  • 중앙에 단계별 hero 설명 + 시각화
  • 이 모드는 환자에게 시뮬레이션 흐름을 설명할 때 유용

12보고서 PDF 다운로드

구성

14용어 사전

30 kHz Ricker pulse
본 시스템에서 source로 사용하는 음향 파동 형태. 중심 주파수 30 kHz의 짧은 wavelet으로, 임상용 초음파(보통 1–20 MHz)보다 훨씬 낮은 주파수입니다.
Vs (전단파 속도)
매질 내 전단파 속도. 조직별로 값이 달라 본 시스템에서는 치아(2.5), 골(1.5), 치은(0.05) 등의 값으로 모델링됩니다. 염증 조직은 약 50% 더 느린 값(0.025)으로 가정.
MPML
Multi-axial Perfectly Matched Layer. 유한 영역 시뮬레이션에서 경계로부터의 반사를 흡수하기 위한 경계 조건.
MCMC
Markov Chain Monte Carlo. Bayesian 사후분포에서 표본을 추출하는 알고리즘. 본 시스템은 Metropolis-Hastings 1000-iter 체인을 사용합니다.
R̂ (R-hat)
MCMC 수렴 진단량. 1에 가까울수록 수렴이 잘 된 것이며, 1.05 이하면 수렴, 1.10 초과면 미수렴으로 해석합니다.
ESS (Effective Sample Size)
유효 표본 크기. 자기상관을 보정한 후 실질적으로 독립인 표본의 수. 본 시스템에서는 고유 posterior state 수로 근사합니다.
posterior mode
사후분포에서 가장 자주 방문된 상태 (최빈값).
MAP (Maximum A Posteriori)
본 시스템에서는 최소 misfit을 달성한 iteration의 상태로 정의.
misfit
관측된 데이터와 모델 예측값 사이의 잔차. 작을수록 모델이 데이터를 잘 설명합니다.
acceptance rate
Metropolis-Hastings 체인에서 제안된 다음 상태가 수락된 비율. 20–60%가 건강한 범위입니다.
credible region (신뢰 영역)
사후분포의 일정 비율(예: 95%)이 포함되는 영역. 빈도주의의 confidence interval과 해석이 다릅니다.
voxel
3D pixel. 본 시스템의 모델은 415 × 308 × 17 voxel 격자이며, 1 voxel ≈ 0.1 mm (근사).
seismogram
수신기에 기록된 시계열 신호. 본 시스템에서는 75,000 timestep × 100 receiver 행렬.
FDI 표기법
ISO 3950 치아 표기 시스템. 첫 자리는 사분면(1=상우, 2=상좌, 3=하좌, 4=하우), 둘째 자리는 중앙선부터의 거리(1–8). 예: 36 = 좌하 첫째 큰어금니.

15한계와 면책 사항

반드시 확인 본 시스템은 PoC 단계의 시뮬레이션 데모이며, 실제 임상 진단 기기로 인증되지 않았습니다.

데이터의 한계

  • 모든 환자 정보, 스캔, 병변은 합성된 가상 케이스입니다
  • 탄성파 시뮬레이션은 사전 계산된 결과이며, 새로운 환자 데이터에 대한 실시간 분석은 지원하지 않습니다
  • 치아·치은·골의 음향 물성치는 문헌값을 사용한 가정이며, 환자별 변동을 반영하지 않습니다

알고리즘의 한계

  • Severity 점수는 잔차의 단일 스칼라이며, 병변의 형태·크기·심도에 대한 정보를 모두 담지 못합니다
  • MCMC 추정의 신뢰도는 R̂ / ESS로 정량화되나, 그 자체가 임상 정확도를 보장하지는 않습니다
  • 현재 시스템은 단일 시점의 횡단면 분석만 다루며, 시간 추적은 케이스 비교를 통한 정성적 확인 수준입니다

임상 의사결정 시 유의 사항

  • 본 시스템의 출력값을 단일 근거로 진료 결정을 내려서는 안 됩니다
  • 치주낭 깊이, 부착 소실, 출혈 여부 등 기존 임상 검진과 병행 검토가 필요합니다
  • 의심 소견이 있을 경우 추가 영상(CBCT 등) 및 생화학적 검사를 통한 확인이 권장됩니다

향후 계획

  • 실제 CBCT + STL 환자 데이터와 DentalSegmentator 분할 결과의 통합
  • In vitro 모델을 이용한 forward 모델 검증
  • Ex vivo 조직에서의 실측 비교
  • 다기관 임상 데이터를 이용한 알고리즘 보정
